UniswapX filling lets market makers compete to execute swaps through permissionless auctions. Use this section to choose a chain, ingest orders, and execute fills safely.

## Choose Your Integration Path

### Ethereum mainnet (quoters + fillers)
Ethereum uses RFQ plus Dutch auctions. Quoters can win temporary exclusivity, and fillers execute when orders are fillable. Start with [Filling on Mainnet](/docs/liquidity/uniswapx/filling/mainnet/filling-on-mainnet). If you want RFQ participation, see [Become a Quoter](/docs/liquidity/uniswapx/filling/mainnet/become-a-quoter).

### Arbitrum (fillers only)
Arbitrum uses Dutch auctions without RFQ. Fillers compete directly onchain. See [Filling on Arbitrum](/docs/liquidity/uniswapx/filling/arbitrum/filling-on-arbitrum).

### Unichain and Base (priority orders)
Priority chains use priority-fee bidding with `PriorityOrderReactor`. See [Filling on Priority Chains](/docs/liquidity/uniswapx/filling/priority-chain/filling-on-op-stack).

## How Filling Works

1. Retrieve open orders from the Uniswap API, or register [webhooks](/docs/liquidity/uniswapx/filling/webhooks) for lower-latency notifications.
2. Decode and evaluate each order with your strategy, inventory checks, and risk controls.
3. Submit `execute` transactions to the target reactor, ensuring required output amounts are met.

## Get Started

1. Join [UniswapX Fillers Chat](https://t.me/UniswapXdiscussion).
2. Implement one chain-specific flow from the guides above.
3. Add webhook support if polling latency becomes a bottleneck.
